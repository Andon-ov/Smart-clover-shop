# Smart Clover Bridge - WordPress Edition 🚀

Този проект представлява интелигентен мост между софтуера за управление на търговски обекти **"Детелина" (Microinvest)** и платформата за онлайн търговия **WooCommerce**.

---

## 🏗 Архитектура (Docker Stack)

Системата е разделена на контейнеризирани услуги за максимална стабилност и лесна поддръжка:

1.  **WordPress (clvr_wordpress):** Хоства WooCommerce магазина.
2.  **MariaDB (clvr_db_wp):** Основна база за WordPress. Гарантира 100% съвместимост с плъгини.
3.  **PostgreSQL (clvr_db_sync):** "Умната" база за синхронизация.
    *   Съхранява таблици за мапване (ID Детелина <-> ID WooCommerce).
    *   Пази история на обработените XML файлове (хеш за предотвратяване на дублиране).
    *   Логове за грешки и трансакции.
4.  **FTP Server (clvr_ftp):** Точка за обмен на данни. Детелина се свързва тук, за да качва и сваля XML файлове.
5.  **Sync Engine (clvr_sync_engine):** Custom скрипт (Node.js), който управлява логиката на интеграцията.

## 🔄 Потоци на данни

### 1. Детелина ➡️ WooCommerce (Импорт)
*   **Номенклатура (PLUDATA):** Когато в Детелина се промени артикул, се генерира XML. Sync Engine го разчита, конвертира от Windows-1251 към UTF-8 и обновява продукта в WooCommerce.
*   **Наличности:** Следи се секцията `<SLPRC>` за актуализиране на количествата в реално време.
*   **Промоции (PROMODATA):** Автоматично активиране на намаления според периоди.
*   **Продажби (POSSALES):** Всяка продажба на физически обект се отразява в онлайн магазина за актуализация на склад.

### 2. WooCommerce ➡️ Детелина (Експорт)
*   **Нови Поръчки:** Sync Engine генерира XML от тип `RDELIV` (Request for Sales, Type 2).
*   **Клиенти:** Първо се изпраща секция `<CUSTOMERS>`, за да се регистрира ЕИК/данни на клиента в Детелина.
*   **Транспорт:** Файловете се качват в `/transfer/out` като `.tmp` и се преименуват на `.xml` след пълно записване.

## 🛠 Инструкции за стартиране

1.  **Конфигурация на средата:**
    Създайте `.env` файл в `/wordpress` със следните променливи:
    ```env
    WP_KEY=ck_your_woocommerce_consumer_key
    WP_SECRET=cs_your_woocommerce_consumer_secret
    ```

2.  **Стартиране на контейнерите:**
    ```bash
    docker-compose up -d
    ```

3.  **Конфигурация на Детелина:**
    Настройте плъгина за експорт в Детелина да сочи към:
    *   **Host:** IP на Docker хоста
    *   **Port:** 21
    *   **User:** `detelina`
    *   **Pass:** `clover_pass`

## 📂 Структура на папките

*   `/transfer/in`: Тук Детелина качва данни.
*   `/transfer/out`: Тук WooCommerce поставя заявки за поръчки.
*   `/transfer/archive`: Успешно обработени файлове се местят тук.
*   `/sync-engine`: Основният код на интеграцията.
*   `/wp-content`: Персистентни данни на сайта (теми, плъгини, снимки).

---

## ⚠️ Управление на грешки

*   Ако Детелина не може да обработи файл, той се преименува на `.err`.
*   Sync Engine следи за `.err` файлове и записва детайлите в PostgreSQL базата за диагностика.
*   Всеки XML файл се хешира (SHA-256) в PostgreSQL, за да се гарантира, че няма да бъде обработен повторно.
